# ARCHITECTURE — стек, модель, математика

## Стек
- **Expo SDK 56** (RN 0.85 + React 19) + **Expo Router**, веб через **react-native-web** — один код
  на web/iOS/Android. Веб-вывод `app.json`: `web.output: "single"` (SPA). `typedRoutes` выключен.
- **NativeWind** (стили) + дизайн-токены в `src/ui/theme.ts` (единый источник).
- **Supabase** (Postgres + Auth + RLS).
- **zod** (валидация), **@tanstack/react-query** (серверное состояние/кэш).
- Бизнес-логика — чистые функции в `src/core/logic`.

## Модель данных: «Сессия целей»
- Жмёшь «Поставить цели» → создаётся **сессия**, отсчёт **от момента создания** (не календарь, не
  регистрация). Сессия = **4 недели = 28 дней**. Одна активная сессия за раз.
- **Goal**: `{ id, sessionId, title, timeframe: day|week|month, target: число, weight: % }`.
  Веса суммируются **до 100% ВНУТРИ своего горизонта** (отдельно day, week, month). Единиц нет.
- **DailyLog**: `{ goalId, date, value }` — факт за день (unique goal_id+date).
- Таблицы: `goal_sessions`, `goals`, `daily_logs`, плюс `profiles` (имя + gamification-статы),
  `achievements`. RLS: каждый видит только своё (через session→goal→log).

## Математика колец (СЕРДЦЕ ПРОДУКТА — не сломать)
Кольца **вложенные, с фиксированными долями**. Все формулы в `src/core/logic/index.ts`:
- **День** = взвешенный прогресс дневных целей за сегодня (100% — дневные цели).
- **Неделя** = `0.6 × (дневная часть) + 0.4 × (недельные цели)`, где
  **дневная часть = Σ(дневное кольцо за 7 дней недели) / 7** (фикс. знаменатель 7). Один идеальный
  день = **1/7 от 60%**. Будущие дни = 0 → неделя наполняется постепенно.
- **Месяц** = `0.8 × (недельная часть) + 0.2 × (месячные цели)`, где
  **недельная часть = Σ(кольцо каждой из 4 недель) / 4**. Одна закрытая неделя = **1/4 от 80%**.
  Месяц завязан на НЕДЕЛИ, не на дни напрямую.
- **Отсутствующая половина = 0 (без перенормировки).** Т.е. без целей на всех уровнях кольца
  недели/месяца не дойдут до 100%. Константы: `WEEK_DAY_WEIGHT=0.6`, `WEEK_OWN_WEIGHT=0.4`,
  `MONTH_WEEKS_WEIGHT=0.8`, `MONTH_OWN_WEIGHT=0.2`.
- **Перевыполнение запрещено**: дневная цель ≤ target/день; недельная ≤ остаток до недельного
  target; месячная ≤ остаток до месячного. Ключевые функции: `computeRings`, `weekRingFor`,
  `goalCardProgress`, `goalCurrent`, `goalMaxToday`, `dayGoalsOn`.

## Цвета горизонтов
День = индиго `#6366F1`, Неделя = голубой `#0EA5E9`, Месяц = фиолетовый `#A855F7`
(в `theme.ts`: `timeframeColor` / `timeframeSoft`).

## Геймификация (`src/features/gamification`)
- **Стрик**: день активен, если дневное кольцо за тот день ≥ 80% (`STREAK_ACTIVE_THRESHOLD`).
- **XP/уровни**: +10 за закрытую за период цель, +5 за идеальный день; кривая `levelFromXp`.
- **Достижения**: first_step, week_streak, perfect_day, session_master. **Heatmap** активности.
- Всё пересчитывается идемпотентно из данных (`engine.ts` + `sync.ts` после каждого лога).

## Auth (`src/features/auth/auth-provider.tsx`)
- Email+пароль, без подтверждения email. `signUpWithPassword` → `completeProfile` (создаёт profile,
  кладётся в query-кэш до навигации). Гейт в `src/app/_layout.tsx` роутит по session/profile/admin.
- Admin-режим: логин `admin`/`1111` → флаг в контексте → роут `/admin` (локальное приложение на
  AsyncStorage + скруббер дней 1..28). Обычные юзеры не затронуты.
- Supabase-клиент: **no-op lock** обязателен (иначе дедлок navigator.locks на вебе). Таймаут 15с на
  auth-вызовы. `detectSessionInUrl` на вебе, `flowType: pkce`.
