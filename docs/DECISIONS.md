# DECISIONS — принятые решения и почему

## Продукт / модель
- **Модель «сессия целей», не программы.** Ушли от первой версии (programs/tasks с периодами
  7/14/30д) к единому списку целей на день/неделю/месяц в рамках 4-недельной сессии. Отсчёт **от
  создания сессии**, не от календаря и не от регистрации (решение пользователя, было 2 пивота).
- **Кольца вложенные, доли 60/40 и 80/20** (не 50/50). Неделя = 60% дни + 40% недельные цели;
  Месяц = 80% недели + 20% месячные цели. Фикс. знаменатели (день=1/7, неделя=1/4), отсутствующая
  половина = 0 без перенормировки. Это явное требование пользователя — **не менять без спроса**.
- **Единиц измерения нет** — только число-цель (убрано по просьбе).
- **Веса — % внутри горизонта, сумма 100%.** Ручной ввод + кнопка «Поровну». (Пользователь думал
  про упрощение весов, но пока оставили как есть.)
- **Одна сессия за раз.** Кнопку создания второй сессии убрали; можно только редактировать текущую
  или удалить и создать новую (с подтверждением, чтобы случайно не сломать прогресс).
- **Перевыполнение запрещено** (кап на target/остаток).
- **Онбординг**: 3 экрана перед постановкой целей (что такое сессия, кольца, откуда проценты).
- **Админ ≠ демо**: под `admin`/`1111` — полноценное локальное приложение (сам вносишь цели/отметки,
  AsyncStorage), единственное отличие — **скруббер дней 1..28** (посмотреть, как заполнится к концу).

## Технические
- **Auth: email+пароль БЕЗ подтверждения email.** Причина: Supabase запретил редактировать
  email-шаблоны без своего SMTP → OTP-код в письмо не положить. Проще и надёжнее для MVP. «Confirm
  email» в Supabase выключен.
- **supabase-js: обязателен no-op `lock`** (`(_n,_t,fn)=>fn()`). Дефолтный `navigator.locks` даёт
  дедлок на вебе — логин-POST 200, но промис висит вечно (были «вечные скелетоны»). Плюс таймаут 15с
  на auth-вызовы как страховка.
- **Роут админа — `app/admin.tsx` (путь `/admin`), НЕ группа `(admin)`.** Группы Expo Router
  прозрачны для пути → `(admin)/index.tsx` коллизировал с главной на `/` (по главной ссылке
  открывалась админка).
- **Web output = "single" (SPA).** `static` (SSR) падал на `window` из auth-хранилища. `vercel.json`
  добавляет SPA-rewrites, иначе 404.
- **typedRoutes выключен** — чтобы строковые пути (`router.push('/(app)/...')`) не ловили типовые
  ошибки.
- **Дизайн: чистый минимал.** Пользователь отверг «тяжёлую» плашку (цветная полоса + залитая
  кнопка) — вернули нейтральные степперы, цвет только в точке и баре. Палитра горизонтов —
  прохладная триада индиго/голубой/фиолет.
