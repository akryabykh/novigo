## Большая недавняя история: «ничего не грузится / вечные скелетоны»
Пользователь несколько раз видел: после логина главная висит на серых скелетонах, логаут не
работает, иногда белый экран. **Диагностика (важно, чтобы не повторять):**
- Сервер/Supabase/сеть — **не виноваты**: авторизованные запросы к REST отвечают за 0.2–0.5с
  (проверено curl + прямой `fetch` в реальном браузере пользователя).
- Причина №1 — **дедлок supabase-js на вебе** через `navigator.locks`: логин-POST возвращал 200,
  но промис не разрешался. Починка: **no-op lock** в конфиге Supabase-клиента
  (`src/core/data/supabase.ts`).
- Причина №2 — **коллизия роутов Expo Router**: `app/(admin)/index.tsx` и главный экран оба вели
  на путь `/` (группы `(...)` не добавляют сегмент), из-за чего по главной ссылке открывалась
  админка. Починка: админ перенесён в `app/admin.tsx` → путь `/admin`.
- Причина №3 (у пользователя) — **закэшированный старый бандл**: даже после фиксов браузер отдавал
  старую сломанную версию. Решение: жёсткая перезагрузка `Cmd+Shift+R`.
- Диагностика делалась через MCP `claude-in-chrome` в реальном браузере пользователя (headless в
  песочнице врал — там сеть к Supabase таймаутила, хотя у пользователя всё быстро).
